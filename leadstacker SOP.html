1. About This SOP

This document defines LeadStacker's internal standards for building, configuring, auditing, and maintaining AI-powered bots for our clients. It covers all three bot platforms currently in our stack:

Closebot

Conversational AI bots for Facebook Messenger, SMS, and Email

CommentGuard

AI-powered Facebook comment management and engagement

Synth

Proprietary AI SMS chatbot system (n8n + Claude API + GHL)

2. Universal Standards (All Platforms, All Clients)

⚠️ NON-NEGOTIABLE RULES
The following rules apply to every bot we build, regardless of client, platform, or campaign type. These must be verified during every QA review.

2.1 The LeadStacker Bot Rules

# Rule Why It Matters
1 Business hours must always reflect actual availability — never say a day is closed if the team is available Incorrect hours cause missed leads and damage trust
2 Bots must never collect payment information, SSNs, or sensitive credentials Legal/compliance risk and platform ToS violation
3 Existing customer complaints must always be routed to official support — never handled by lead bots Confusing support with sales destroys the lead experience
4 Opt-out requests (STOP, UNSUBSCRIBE, NO THANKS, etc.) must be honored immediately and permanently Legal requirement under TCPA/CAN-SPAM
5 All bots must have proper abuse detection active Prevents abuse and fake leads from wasting budget
6 Every bot edit must be saved AND published — a draft is not live Common cause of "why isn't the bot working" tickets
7 Re-open Settings after every publish to verify changes saved correctly Platforms can silently revert changes on crash
8 Bot identity (AI vs. human) must follow client's explicit preference Some clients require AI disclosure; others prefer human persona
9 All offer details must be verified with client before going into any bot Incorrect offers create chargebacks and complaints
10 Maximum 3 follow-up attempts per lead across all channels combined More than 3 hurts deliverability and annoys leads

2.2 Naming Convention

Consistent naming makes it easy to identify bots at a glance. Follow this format:

Format: [Client Name] Bot ([Channel]) [Variant]

Examples:
• Collins Mobile Bot (FB)
• Collins Mobile Bot (SMS/Email)
• Collins Mobile - Sales Rep (On-Hour)
• Invention Connect - Synth SMS

2.3 Persona Standards

Every bot needs a named persona that defines tone, voice, and identity:

Setting Standard
Name Common first name (Jamie, Sam, Alex) — agreed with client
Role Description Clear and specific: "[Client] Sales Rep" — not generic
Tone Always: Friendly + Concise. Add Sincere/Realistic for sales.
First Person Always use "we"/"our" — never "they"/"their"
Reply Length Default: 2–3 lines per message. Keep conversational.
AI Disclosure Follow client preference. Document in config.

5. Synth — AI SMS Chatbot System

🚀 LeadStacker Proprietary Technology
Synth is our custom-built AI SMS chatbot system combining n8n automation, Claude AI, and GoHighLevel for intelligent lead conversations.

5.1 What Synth Does

Synth enables real-time, contextual SMS conversations with leads:

  • Texts new leads immediately when they come in from ads or forms
  • Has real conversations — handles objections, answers questions, remembers context
  • Drives leads toward conversion (forms, bookings, calls)
  • Tags and tracks everything automatically in GHL
  • Multi-tenant — one system serves all LeadStacker clients
Important: This is NOT a simple autoresponder. Synth reads full conversation history and generates contextual responses via Claude API.

5.2 Synth Architecture Overview

Facebook Ad / Website Form
    ↓
GHL Sub-Account (client's account)
    ↓ (webhook fires, includes locationId)
n8n — ONE pair of master workflows
    ↓ (looks up client config by locationId)
Claude API — Jamie persona, built dynamically
    ↓ (returns: SMS reply + JSON actions)
n8n executes actions (tags, updates)
    ↓
GHL sends SMS back to lead
    ↓
Lead replies → loops back through chain

5.3 Tech Stack Components

Layer Tool Purpose
CRM + SMS GoHighLevel (GHL) All client data, conversations, pipelines
Automation n8n Orchestrates everything — runs on LeadStacker VPS
AI Claude API Haiku for volume, Sonnet for high-value leads
Lead Source Meta Lead Ads + Landing Pages Triggers GHL contact creation

5.4 Key Features in Synth v2

1. Lead Scoring (1–10 Intent)

Claude scores each lead 1–10 on intent level every turn:

  • Score ≥ 8 → synth_high_intent tag + GHL alert
  • Score 4–7 → synth_medium_intent tag
  • Score 1–3 → synth_low_intent tag

2. Sentiment Detection → Auto-Escalation

Claude returns sentiment field (positive/neutral/negative). If negative sentiment detected:

  • Auto-adds synth_escalate tag
  • Auto-adds synth_negative_sentiment tag
  • Human team gets notification for follow-up

3. Booking Confirmation Loop

When Jamie sends conversion link:

  • Workflow starts 3-hour wait timer
  • After 3 hours, rechecks contact tags in GHL
  • If not completed → sends one friendly follow-up nudge
  • Nudge sent ONCE only

4. First-Reply Speed Tracking

Timestamp of first SMS logged as GHL contact note showing "Speed to contact: Instant (automated)" — visible to sales team.

5.5 Synth Tag Structure

Tag Meaning
synth_active Synth is managing this lead
synth_intro_sent First message sent
synth_engaged Lead replied and is in conversation
synth_interested Lead expressed genuine interest
synth_isa_sent Conversion link sent to lead
synth_isa_complete Lead confirmed completion
synth_high_intent Lead score 8+ — flag for human outreach
synth_escalate Needs human follow-up
do_not_contact STOP received — never contact again

5.6 Setting Up Synth for a New Client

Prerequisites:
• Client has active GHL sub-account
• GHL Agency Token configured in n8n
• Claude API key configured in n8n
• Base prompt loaded in n8n Variables

Step-by-Step Setup

  1. Get Client Information: GHL Location ID, offer details, business hours, conversion link
  2. Create Client Config: Fill in all required fields in config template
  3. Add Config to n8n: Paste into BOTH workflow Code nodes (New Lead + Inbound Reply)
  4. Set Up GHL Automation: Create "Synth — New Lead Trigger" workflow in client's GHL
  5. Set Up GHL Webhook: Add InboundMessage webhook pointing to n8n
  6. Test End-to-End: Create test contact, verify SMS flow
  7. Document: Add to client inventory and change log

5.7 Synth QA Checklist

  • Client config added to BOTH n8n workflows
  • locationId is correct (copied exactly from GHL)
  • Business hours match client's actual availability
  • Conversion link is correct and tested
  • GHL Automation created and published
  • GHL Webhook configured with correct URL
  • Test SMS received within 2 minutes
  • Reply triggers contextual response
  • Tags applied correctly in GHL
  • STOP keyword triggers opt-out

6. GoHighLevel (GHL) Integration Standards

All LeadStacker bots integrate with GoHighLevel as the central CRM and lead management system.

6.1 GHL Sub-Account Structure

  • Each client has their own GHL sub-account under LeadStacker agency
  • All client lead data lives in their sub-account
  • LeadStacker team has admin access to all sub-accounts
  • Clients have user-level access to their own sub-account

6.2 GHL Smart Views for Bot Management

Create these Smart Views in each client's GHL sub-account:

View Name Filter Purpose
🤖 Active Bot Leads synth_active OR cb_active All leads currently managed by bots
🔥 Hot Leads — Follow Up synth_high_intent High-intent leads needing human outreach
⚠️ Escalated synth_escalate Leads flagged for human attention
📋 Forms Sent — Pending synth_isa_sent, NOT synth_isa_complete Received form but haven't completed
✅ Conversions synth_isa_complete Completed conversions
🚫 Do Not Contact do_not_contact Opted-out leads
